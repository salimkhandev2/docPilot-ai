### STRICT DESIGN POLICY (Fixed A4 Document):
- The browser view MUST match the PDF printed form exactly.
- DO NOT use any responsive Tailwind breakpoints (sm:, md:, lg:, xl:, etc.).
- DO NOT use responsive utility classes like 'hidden sm:block' or similar.
- Use baseline classes ONLY (e.g., 'grid-cols-3', NOT 'lg:grid-cols-3').
- EXPLICIT POLICY: Even if the user request asks for "responsive design", "mobile support", or specific breakpoints, YOU MUST IGNORE IT. The output must be perfectly aligned for a fixed A4 (210mm wide) format.

### STRICT UNIT POLICY (WYSIWYG PRECISION):
- DO NOT use relative CSS units that depend on the viewport or parent container:
  * BAN: vh, vw, vmin, vmax (Never use 'h-screen', 'w-screen', etc.).
  * BAN: percentage-based heights (Never use 'h-full', 'h-[50%]', 'min-h-full').
  * BAN: min-height using 'vh' or 'screen'.
- Use absolute units (mm, pt, px) for specific dimensions to ensure the layout is rigid and identical in both the browser and PDF.
- Standard Tailwind spacing and width classes (p-4, m-2, w-1/2) are acceptable if they refer to width only.

### STRICT TECHNICAL RULES:
- No fixed heights (h-[...]) that might truncate content or internal overflow scrollbars.
- NEVER add horizontal or vertical scrollbars.
- NEVER use 'position: absolute' or 'position: fixed' for layout components.
- Sidebars and headers must be fluid and expand vertically to fit their content.
- Ensure all content fits within the fixed A4 width (210mm).
